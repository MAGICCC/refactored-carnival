Prerequisite system.de

Bevor Sie mailcow: dockerized ausfhren, sollten Sie einige Voraussetzungen berprfen:

Achtung

Versuchen Sie nicht, mailcow auf einem Synology/QNAP-Gert (jedes NAS), OpenVZ, LXC oder anderen Container-Plattformen zu installieren. KVM, ESX, Hyper-V und andere vollstndige Virtualisierungsplattformen werden untersttzt.

Info

  • mailcow: dockerized erfordert, dass einige Ports fr eingehende Verbindungen offen sind, also stellen Sie sicher, dass Ihre Firewall diese nicht blockiert.
  • Stellen Sie sicher, dass keine andere Anwendung die Konfiguration von mailcow strt, wie z.B. ein anderer Maildienst
  • Ein korrektes DNS-Setup ist entscheidend fr jedes gute Mailserver-Setup, also stellen Sie bitte sicher, dass Sie zumindest die basics abgedeckt haben, bevor Sie beginnen!
  • Stellen Sie sicher, dass Ihr System ein korrektes Datum und eine korrekte Zeiteinstellung hat. Dies ist entscheidend fr verschiedene Komponenten wie die Zwei-Faktor-TOTP-Authentifizierung.

Minimale Systemressourcen¶

Bitte stellen Sie sicher, dass Ihr System mindestens ber die folgenden Ressourcen verfgt, um problemlos zu laufen:

Ressource Minimale Anforderung

RAM Minimum 6 GiB + 1 GiB Swap (Standardkonfiguration)
Festplatte 20 GiB (ohne Emails)
Architektur x86_64, ARM64

Nicht untersttzte Plattformen

OpenVZ, Virtuozzo und LXC

Arbeitsspeicher¶

ClamAV und die FTS-Engine (Flatcurve) knnen viel RAM nutzen, lassen sich aber ber die Parameter SKIP_CLAMD=y und SKIP_FTS=y in der mailcow.conf deaktivieren.

mailcow ist eine umfassende Groupware mit zahlreichen Features wie Webserver, Webmailer, ActiveSync, Antivirus, Antispam, Indexierung, Dokumentenscanner, Datenbank und Cache, weshalb es mehr Ressourcen bentigt als ein einfacher MTA.

Ein einzelner SOGo-Worker kann ~350 MiB RAM belegen, bevor er geleert wird. Je mehr ActiveSync-Verbindungen Sie verwenden mchten, desto mehr RAM wird bentigt. In der Standardkonfiguration werden 20 SOGo-Worker erzeugt.

Beispiele fr die RAM Planung¶

Ein Unternehmen mit 15 Smartphones (EAS aktiviert) und etwa 50 gleichzeitigen IMAP-Verbindungen sollte 16 GiB RAM einplanen.

6 GiB RAM + 1 GiB Swap sind fr die meisten privaten Installationen ausreichend, whrend 8 GiB RAM fr ~5 bis 10 Benutzer empfohlen werden.

Im Rahmen unseres Supports knnen wir Ihnen bei der korrekten Planung Ihres Setups helfen.

Firewall & Ports¶

Bitte berprfen Sie, ob alle Standard-Ports von mailcow offen sind und nicht von anderen Anwendungen genutzt werden:

ss -tlpn | grep -E -w '25|80|110|143|443|465|587|993|995|4190'
# oder:
netstat -tulpn | grep -E -w '25|80|110|143|443|465|587|993|995|4190'

Vorsicht

Es gibt einige Probleme mit dem Betrieb von mailcow auf einem Firewalld/ufw aktivierten System.
Sie sollten es deaktivieren (wenn mglich) und stattdessen Ihren Regelsatz in die DOCKER-USER-Kette verschieben, die nicht durch einen Neustart des Docker-Dienstes gelscht wird.
Siehe diese (blog.donnex.net) oder diese (unrouted.io) Anleitung fr Informationen darber, wie man iptables-persistent mit der DOCKER-USER Kette benutzt.
Da mailcow im Docker-Modus luft, haben INPUT-Regeln keinen Effekt auf die Beschrnkung des Zugriffs auf mailcow.
Verwenden Sie stattdessen die FORWARD-Kette.

Wenn dieser Befehl irgendwelche Ergebnisse liefert, entfernen oder stoppen Sie bitte die Anwendung, die auf diesem Port luft. Sie knnen mailcows Ports auch ber die Konfigurationsdatei mailcow.conf anpassen.

Wichtig fr Hetzner Firewalls¶

Ich zitiere https://github.com/chermsen ber https://github.com/mailcow/mailcow-dockerized/issues/497#issuecomment-469847380 (DANKE!):

Fr alle, die mit der Hetzner-Firewall zu kmpfen haben:

Port 53 ist in diesem Fall fr die Firewall-Konfiguration unwichtig. Laut Dokumentation verwendet unbound den Portbereich 1024-65535 fr ausgehende Anfragen. Da es sich bei der Hetzner Robot Firewall um eine statische Firewall handelt (jedes eingehende Paket wird isoliert geprft) - mssen die folgenden Regeln angewendet werden:

Fr TCP 

SRC-IP: ---
DST-IP: ---
SRC-Port: ---
DST-Port: 1024-65535
Protokoll: tcp
TCP-Flags: ack
Aktion:      Akzeptieren

Fr UDP 

SRC-IP: ---
DST-IP: ---
SRC-Port: ---
DST-Port: 1024-65535
Protokoll: udp
Aktion:      Akzeptieren

Wenn man einen restriktiveren Portbereich anwenden will, muss man zuerst die Konfiguration von unbound ndern (nach der Installation):

{mailcow-dockerized}/data/conf/unbound/unbound.conf: 

outgoing-port-avoid: 0-32767

Nun knnen die Firewall-Regeln wie folgt angepasst werden:

[...]
DST Port: 32768-65535
[...]

Hetzner Cloud (und wahrscheinlich andere)¶

Prfen Sie /etc/network/interfaces.d/50-cloud-init.cfg und ndern Sie die IPv6-Schnittstelle von eth0:0 auf eth0:

# Falsch:
auto eth0:0
iface eth0:0 inet6 static
# Richtig:
auto eth0
iface eth0 inet6 static

Starten Sie die Schnittstelle neu, um die Einstellungen zu bernehmen. Sie knnen auerdem die cloud-init Netzwerknderungen deaktivieren.
